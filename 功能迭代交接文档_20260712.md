## 1. 本轮已落地功能

### 1.1 商品列表页卡片（最终定稿，`ProductScreen` + `UserProductSection`）

经过多轮截图迭代，现状如下（与旧文档描述不同，**以本节为准**）：

- **用户分区**：`UserProductSection` 标题行仅 `nickname`（主色粗体）+ 灰字「清空」按钮。
- **类型分组**：框内按商品**四大类**排序分段，顺序固定 `上半身 → 下半身 → 全身 → 饰品`，未匹配类型单独成段；段间 `spacedBy(12.dp)` 空白分隔，**无类型标题文字**。
  - 分类依据 `resolveCategory()`：商品名**去掉颜色前缀**后按 `upperClothingOptions` 等列表前缀匹配（短袖/长袖都归上半身，与颜色无关）。
  - `clothingOptions` 四个列表在 `ProductScreen.kt` 由 `private` 改为 **`internal`**（:43 起），供同包 `UserProductSection` 识别类型。
- **网格**：固定 **3 列**（`colCount = 3`），`chunked(3)` 分行；窄屏每卡约 100dp。
- **小卡片 `ProductMiniCard`**（左信息 + 右操作）：
  - 左侧：商品名（14sp，**加 `Ellipsis` 防溢出**）+ 价格（12sp）。
  - 右侧：**必买 `Checkbox`**（24dp，勾选红色 `#E24B4A`）——**已用 Checkbox 替代原 Switch**（Switch 内部 clickable 会吞拖拽手势，详见 §2）。
  - 背景按类型上色（方案 A 马卡龙浅色，`categoryBackground` map）：上半身浅蓝 `#E0F2FE`、下半身浅绿 `#DCFCE7`、全身浅黄 `#FEF9C3`、饰品浅紫 `#F3E8FF`；边框同色系深一档（`categoryBorder`）。
  - **必选优先级最高**：红底 `#FEE2E2` + 红框 `#E24B4A`，**覆盖**类型色，无文字标签。
  - 未匹配类型商品：白底 + 浅灰边。
- **左滑删除**（手势挂在前景 Card 上）：
  - 结构：`Box`(容器) > 底层红底删除区 `Box(matchParentSize + clickable→onDelete)` + 前景 `Card(offset + draggable + clickable)`。
  - 横向 `draggable` 累加 `offsetX`（限制在 `-56dp~0`）；松手按 50% 阈值决定开/关。**两个方向都能滑**（左开右关）。
  - 已滑开时：点卡片内容收起；未滑开时：点卡片 → 编辑弹窗（同层 `clickable` 与 `draggable` 共存，互不抢）。
  - 删除区图标点击才删（防误删）。
- **用户列表抽屉 `ModalDrawer`**（`ProductScreen.kt`）：
  - `gesturesEnabled = false` → **任意位置横滑不再打开抽屉**（原默认行为覆盖整个内容区，导致屏幕中间滑动也能划出，属误触）。
  - 左上角菜单按钮 = **toggle**（`isOpen ? close : open`），禁拖拽后仍能开关。
  - `UserDrawerContent` 新增 `onClose` 参数 + 顶部 **X 关闭按钮**（`ProductScreen:87` 传入 `drawerState.close()`）。禁用拖拽后遮罩点击关闭也失效，必须有显式关闭入口。

### 1.2 多用户凑单「现状 vs 全局」公平对比

（上一会话已落地，本轮未动，仍有效）

核心诉求：跨用户凑单时，原算法只最大化总省钱、无公平性维度，导致某用户被多分摊。

- **`DiscountCalculator.kt`**：新增 `enum class FairnessStrategy { NONE, FAIR }`；`calculateBestCombination(... fairness)` 透传；新增 `ownerFairness` 字段（平局时 `isBetterThan` 用 FAIR 优先让高价用户多留优惠，**绝不牺牲总省钱**）。
- **`CalculatorViewModel.kt`**：新增 `MultiUserSolution` + `_candidateOptions`/`_selectedCandidateKey`；`selectCandidate` 切换应用；分摊逻辑抽 `buildNonStackableShares`。产出两套候选：现状（两阶段）vs 全局（`multiUserMode=false, fairness=FAIR`）。
- **`CalculatorScreen.kt`**：`CandidateCompareCard`（:382 定义，:220 调用）双卡并排，选中高亮切方案。

### 1.3 单用券分摊说明改「按用户视角」（`CalculatorScreen.kt`，`!couponMode` 分支）

设计目标：原「按券/组」视角（每张券一张组卡，用户要在多张卡里找自己）翻转成「按用户」视角，更符合用户心智。

**结构（自上而下）**：
1. **每个勾选用户一张浅蓝块**（`PerUserCouponShareCard`，底 `#E8F4FC` + 边 `#B5D4F4`，对齐叠加券 `PerUserShareCard` 视觉）：
   - **块头**：紫色头像圈（昵称首字）+ 昵称 + 右侧大号红字「实付」。**实付口径 = 自达标 + 单独付 + 凑单分摊总和**（scheme A：块头给总数最直观，配小字「含凑单分摊 -¥X，见底部凑单组合」）。
   - **块内仅自达标组 + 单独付**：
     - `SelfGroupMiniCard`（白底单人组卡）：券名（紫 `#534AB7`）+ 单人组绿/`AppBlue`标签 + 该用户商品 + 底部「商品总价（删除线，移到应付**上方**）+ 应付」。
     - `SeparatePaymentCard`（灰底 `#F5F5F5`）：`ownerId==u` 且不在任何券的商品 → 原价结算。
   - 参与凑单但无自达标组的用户：块内显示「无自达标组 · 参与凑单（见底部）」，块头仍显示其凑单分摊实付。
2. **底部绿色「凑单组合（多人拼券）」区**（`#E8F8EE` 底）：所有凑单组（≥2 人）和空归属填充组下沉到此，**复用 `GroupCombinationCard`**（加 `backgroundColor` 参数，pool 区传浅绿），完整列出组内所有人商品 + 组内按原价占比分摊。组合全貌集中一处。

**组内分摊行细节**（`GroupCombinationCard` 内）：
- 每人应付：`14sp` **加粗红色**（`AppRed` + `Bold`）。
- 每人优惠金额 `-¥X`：`13sp` 加粗**黑色**（`AppInk`，用户要求去红）。
- 占比进度条：`AppLine` 浅底 + `AppBlue` 填充。

**代码清理**：删除死代码 `RemainingProductsCard`（未用券商品现归到用户块「单独付」）；`displayColor` 废弃的 `_coupons` 参数加 `@Suppress` 消除遗留 warning。

**对账校验**：三类（自达标组 / 凑单组 / 单独付）对商品**互斥覆盖**，无重复计算；所有用户块「实付」之和 = `merged.finalPrice`。

## 2. 关键设计决策（必看，避免重蹈覆辙）

1. **Compose 手势坑（最重要）**：
   - `swipeable`/`rememberSwipeableState` 在 **1.5.4 仍在 `androidx.compose.material` 包**（移到 `foundation.gestures` 是 1.6+），第一次用错路径编译挂。
   - `swipeable`（外层 Box）+ 子 `clickable`/`detectTapGestures`（前景 Card）= **父子手势竞争**，子元素霸占指针事件，父收不到拖拽 → 划开后关不掉。最终改用同元素的 `draggable`+`clickable`（官方推荐共存模式）。
   - **手势必须挂在前景 Card 上、不能挂外层 Box**：前景 Card 左滑 offset 后超出 Box 边界，那块区域收不到外层手势 → 划不回。挂 Card 上则手势区域跟随平移。
   - **必买 `Switch` → `Checkbox`**：Switch 内部 clickable 会从它起手吞拖拽，父 `draggable` 收不到。换小 Checkbox 后死区大幅缩小（24dp vs 52dp）。
2. **Drawer 误触**：`ModalDrawer` 默认 `gesturesEnabled=true` 拖拽检测覆盖全内容区（不限于左边缘），屏幕中间横滑会划出抽屉。禁用拖拽后**遮罩点击关闭也失效**，必须给用户列表加显式 X 关闭按钮，且菜单按钮改成 toggle（否则打开后被遮挡点不到）。
3. **「全局」候选必须用 `multiUserMode = false`**（公平对比功能）：误用 `true` 会砍掉自组商品、总省钱反降。
4. **公平性 = 纯 tie-break**，只在总省钱平局时改优惠归属，永不因公平少省钱。
5. **删文字 ≠ 删结构**：类型分组只删标题文字、保留分段布局。
6. **Compose 版本限制**：1.5.4 无 `Modifier.dragAndDropSource/Target`（1.6.0+）。
7. **不确定就问，别乱动手**（用户原话）：改动涉及多处/有歧义先对齐，简单任务直接给结果。
