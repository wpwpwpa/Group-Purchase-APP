### 1.2 多用户凑单「现状 vs 全局」公平对比
核心诉求：跨用户凑单时，原算法只最大化总省钱、无公平性维度，导致某用户被多分摊（例：用户1=[4,99,49]、用户2=[96]，满100减20，原逻辑易把 4 锁死在用户1自组，使 49+96 凑单、用户2 多花约 6 元）。

落地三文件改动：
- **`DiscountCalculator.kt`**
  - 新增 `enum class FairnessStrategy { NONE, FAIR }`（companion 内）。
  - `calculateBestCombination(... fairness: FairnessStrategy = NONE)` 透传公平策略。
  - 新增 `ownerFairness` 字段（可加性公平分：`discount × 最高价商品所属用户的占比`），平局时 `isBetterThan` 用 FAIR 优先让高价用户多留优惠。**绝不牺牲总省钱**（只在 `netSavings` 平局时介入）。
  - 已删除误加的 `calculateFairCandidates`（它用 `multiUserMode=true` 会把自组商品砍掉、总省钱反降，是错误路径）。
- **`CalculatorViewModel.kt`**
  - 新增数据类 `MultiUserSolution(key, title, combined, shares, perUser)`。
  - 状态：`_candidateOptions` / `candidateOptions`（两套方案）、`_selectedCandidateKey` / `selectedCandidateKey`。
  - `selectCandidate(key: String)`：用户在「现状 / 全局」间切换应用，写入 `_combinedSolution`/`_shares`。
  - 分摊逻辑抽成 `buildNonStackableShares(...)`，现状与全局共用，**分摊口径一致**。
  - 产出两套候选：
    - **现状** = 原两阶段（先 perUser 自组 + 后 crossPool 凑单）合并解。
    - **全局** = `calculateBestCombination(scoped, fillProducts, useFill, multiUserMode = false, fairness = FAIR)`。
- **`CalculatorScreen.kt`**
  - 结果区顶部新增 `CandidateCompareCard`（:382 定义，:220 调用）：现状 / 全局双卡并排，显示各自应付总额 + 各人应付，选中高亮、点击切 `selectCandidate`。下方汇总/分摊 Tab 随选中方案切换。

## 2. 关键设计决策（必看，避免重蹈覆辙）

1. **「全局」候选必须用 `multiUserMode = false`**。若误用 `true`，其 buildCandidates 过滤会砍掉自组商品（如 4+99），总省钱从 40 掉到 20，比现状更差。**这是最容易踩的坑**。
2. **两阶段「先自组后凑单」是现状方案的根基**，有意保留未动。公平优化只在「全局」路径生效（全局把全部勾选商品一起分组）。
3. **公平性 = 纯 tie-break**，只在总省钱平局时改变优惠归属，用户永不因公平而少省钱。
4. **删文字 ≠ 删结构**：用户两次强调「删类型标题文字」是只删标签文字、保留按类型分段布局；不要理解为「合并成连续混排 grid」。
5. **Compose 版本限制**：1.5.4 **没有** `Modifier.dragAndDropSource/Target`（那是 1.6.0+）。要做拖拽只能用 `pointerInput { detectDragGestures }` + `onGloballyPositioned` 命中检测，或升级 Compose。
6. **不确定就问，别乱动手**（用户原话）：改动涉及多处/有歧义时先对齐，简单明确任务直接给结果。

## 4. 编译与环境注意

- **JAVA_HOME 必须设**：指向 Android Studio 自带 JBR。本机为 `C:\Program Files\Android\Android Studio\jbr`。不设置则 gradle 找不到 java。
- **离线编译**：`./gradlew compileDebugKotlin --offline`（或 `gradlew.bat`）。
- **沙箱限制（你本地不受影响）**：`--rerun-tasks` 会在 `processDebugResources`（AAPT2）报「failed to start daemon / UCRT 缺失」，属沙箱缺系统库，**与代码无关**。验证 Kotlin 编译正确性用 `compileDebugKotlin --offline`（已验证 BUILD SUCCESSFUL）。
- 提交记录：`9f88093`（商品列表 UI 重做 + 多用户），分支 `feature/product-list-ui-redesign` 已推远程。`.workbuddy/` 不入库（本地工作记忆）。
